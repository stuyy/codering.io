How do events work?

We will host events periodically. They can either be weekly, bi-weekly, or monthly. On the day of the event, the repository will be public and there will be a prompt that explains all of the tasks & features required for the project.

Every contribution will grant the author a certain amount of points. For example, if you create a pull request, you will be given 1 point. If your pull request gets closed, you will lose the point. If your pull request gets merged, you will receive 2 points.

 How are events held?

All events will be hosted using GitHub. All pull requests and contributions made are on GitHub. Please be sure to sign up for GitHub, it's free.

 Are there any prerequisites?

We ask that you have a basic understanding of programming first, as the projects will require you to be able to actually program. Not having an understanding of programming will hinder your ability to make contributions.

 What kind of projects will we be able to work on?

Projects will all be listed on GitHub. They can vary between either a Web API built with Express.JS, a React Application that fetches & renders data, or a Full-Stack application that allows user authentication.

 Are we allowed to form teams?

Not at the moment.

 Are there any rewards or prizes?

Yes! Rewards are based on different factors, such as the event size and the duration. If we have more contributors than we expect, we will throw in more rewards. If the event spanned out for a longer duration, the prize pool will be bigger.

 What happens if I don't win? What happens to my points?

Your points will accumulate over time. At the moment there are future plans to create a virtual shop that allow you to redeem those points for rewards.
